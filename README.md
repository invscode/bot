这是一个用在polymarket上的套利机器人。当然现在的功能很有限，主要是为了给招人的单位看看我的水平够不够格混口饭吃。
已经做到的功能：
检测这个事件："polymarket上的四种币之一：BTC,ETH,SOL,XRP 的15分钟内会上涨吗?"可以看实时的订单簿买一到买十，卖一到卖十，并且计算中间价，价差，和10秒里的波动。
现在只做了一个交易策略：检测是不是有价格闪崩，如果有就买进。在止盈和止损点卖出。
使用方法：
1.设置好下面的配置文件。
.env：
POLY_PRIVATE_KEY=0x100bbe4256a71611226723747e1551f0c6f137786e8509aa833892b732898a89
POLY_PROXY_WALLET=0xE62687bdbF095Db38014557b446970C1fa379B19

src\config.yaml:
safe_address: "0xE62687bdbF095Db38014557b446970C1fa379B19"（最少要有这一条）

apps/flash_crash_config.yaml：（下面是示例）
coin: ETH              # 币种
size: 5.0              # 交易金额 (USDC)
drop: 0.30             # 下跌阈值 (概率变化)
lookback: 10           # 回溯窗口 (秒)
take_profit: 0.10      # 止盈 (美元)
stop_loss: 0.05        # 止损 (美元)
debug: false           # 是否启用调试日志

apps\orderbook_config.yaml:（下面是示例）
coin: BTC

3.目前的性能：
代码的整体结构适合扩展更多的交易策略，比如荷兰式套利等。现在代码里只支持在一个方向上只开一个仓，也就是只能开仓买进一笔，接着就不能再买进了。实际上polymarket是允许买进多笔的。
代码的和服务器的通信可能有问题，早先我开发的时候，碰到一个收不到websocket传来的行情的问题，可是以前是收得到的。调试了一会以后才发现原因是polymarket封了我的账号，大概是因为我挂着VPN又搞这搞那，触发了polymarket服务器的安全机制。我这开发也就没法接着做了。
